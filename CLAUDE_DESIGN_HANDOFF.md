# Claude Code 向けデザイン提案 Handoff

## 1. 今回の目的

このプロジェクトは、**「健康経営優良法人2026」認定取得を目指す企業向けの事例検索＆セルフチェックサイト**です。

今回は **実装を進めることが目的ではなく、デザイン面の改善提案を Claude Code（fable5）に出してもらうこと** が目的です。

そのため、まずは以下をお願いしたいです。

- 現状UIの良い点・気になる点の整理
- デザイン方向性の提案
- どこをどう変えると見やすく、信頼感があり、使いやすくなるかの提案
- まだコードは書かず、**提案と改善方針だけ** を返す

## 3. サイトの役割

利用者は、健康経営優良法人の認定取得を目指す企業担当者です。

このサイトには大きく2つの役割があります。

1. 認定企業の取り組み事例を探せること
2. 自社の取り組み状況をセルフチェックできること

ゴールは、単に見た目を派手にすることではなく、

- 信頼感がある
- 専門的に見える
- 情報が探しやすい
- セルフチェックが迷わず進められる
- 問い合わせにつながる

という体験にすることです。

## 4. 現在の画面構成

### タブ1: 事例検索

- 業種、評価項目、キーワードで事例を絞り込み
- 認定企業27社の取り組み事例を表示
- 展開すると詳細や出典が見える
- フロム・シェフで支援可能な項目には問い合わせ導線あり

### タブ2: セルフチェック

- 29項目の評価項目に対して「はい / いいえ / わからない」で回答
- 進捗バーあり
- 結果画面で充足率・カテゴリ別状況・問い合わせ導線を表示

### 共通要素

- ヘッダーに2タブ切替
- フッターに問い合わせCTA
- 日本語サイト

## 5. 今の実装前提

- Vite 8 + React 19
- JavaScript
- バックエンドなし
- データは静的ファイル管理
- スタイルは主に **インライン style**
- 既存データはそのまま活かしたい

## 6. 触らないでほしい前提

今回は「デザイン提案」が目的なので、以下は勝手に変えない前提でお願いします。

- 事例データの意味や構造
- 評価項目の意味や件数
- `canSupport` の対象項目
- 問い合わせ先メールアドレス
- サイトの基本目的
- 日本語UIであること

## 7. 特に見てほしい観点

以下の観点で、率直にレビューしてほしいです。

- 第一印象が少し素朴すぎないか
- BtoB向けの信頼感・整理感が足りない部分はどこか
- 情報量が多い画面でも読みやすくできるか
- フィルタやセルフチェックの操作がもっと直感的にできないか
- CTA が自然に目に入るか
- モバイルでも窮屈に見えないか
- 色・余白・文字サイズ・カード使いが単調ではないか

## 8. 期待するアウトプット

Claude Code には、まず次の形で返してほしいです。

1. 現状デザインの課題整理
2. デザイン改善の方向性を 2〜3案
3. その中で最もおすすめの1案
4. その案を採用する場合の画面別変更方針
5. 実装に入るなら、どのファイルをどう触るかの見立て

大事な点:

- **この段階ではコード変更はしない**
- 提案は、非エンジニアにもわかる言葉で説明する
- 見た目だけでなく、使いやすさも含めて提案する
